javascript设计模式与开发实践-第一部分

发表于 2017-07-18 | 更新于 2017-07-20 | 分类于 JavaScript

面向对象

JavaScript 是动态类型语言
静态类型： 在编译时便已确定变量的类型。 特点：可靠，但繁琐
动态类型： 变量类型要到程序运行的时候，待变量被赋予某个值之后，才会具有某种类型。 特点：简洁灵活，但易出错

example：鸭子类型 关注 HAS-A 而非 IS-A

面向对象的三大特性：

  • 多态：将“做什么”与“怎么做”分离，静态类型语言通过抽象类和接口来实现，JS 没有约束，比较灵活
  • 封装：无原生支持，可以通过闭包实现私有变量
  • 继承：采用原型继承

this、call 和 apply

JavaScript 的 this 总是指向一个对象，而具体指向哪个对象是在运行时基于函数的执行环境动态绑定的，而非函数被声明时的环境。

常用的this指向：

  • 作为对象的方法调用——指向该对象
  • 作为普通函数调用——window
  • 构造器调用——该对象
  • Function.prototype.call 或 Function.prototype.apply调用。

call、apply区别： 参数不同

用途：

  1. 改变this指向；
  2. Function.prototype.bind；
  3. 借用其他对象方法

闭包和高阶函数

闭包

闭包的形成与变量的作用域以及变量的生存周期密切相关
变量的作用域： 就是指变量的有效范围。全局变量和局部变量(函数内var关键字声明)。

生存周期：

  • 全局变量的生存周期当然是永久的，除非我们主动销毁这个全局变量。
  • 在函数内用 var 关键字声明的局部变量，当退出函数时，这些局部变量即失去了它们的价值，它们都会随着函数调用的结束而被销毁。

闭包就是内部函数引用了外部函数的局部变量，当外部函数执行完毕时，局部变量不被销毁。

闭包的作用

  1. 封装变量
  2. 延续局部变量的寿命

高阶函数

高阶函数是指至少满足下列条件之一的函数。

  • 函数可以作为参数被传递；
  • 函数可以作为返回值输出。

JS 中函数是一等对象，可以作为参数被传递，也可以作为返回值输出。 显然满足高阶函数的条件。

使用场景

参数传递：

  • 回调函数（异步）
  • Array.prototype.sort

返回值输出：

  • 判断数据类型
  • getSingle（单例模式）
